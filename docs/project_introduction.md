# 宿舍生活用品智能小助手：技术介绍

本项目将宿舍用品管理拆成两个不同问题：先根据名称和描述识别物品类别，再根据库存、消耗、有效期与破损状态判断风险。界面使用 Streamlit，支持单件预测、批量清单、解释与反馈记录。

## 1. 类别识别

优化后的类别管线包括：

1. 字符 `2-5 gram` TF-IDF，增强对品牌、缩写和未登录词的适应能力。
2. `jieba` 中文词组 `1-2 gram` TF-IDF，保留词义组合。
3. 卡方特征筛选和逻辑回归，输出 7 类概率。
4. 仅对纸巾、药品、数据线等高精度宿舍关键词执行领域规则覆盖；其余输入由模型判断。

训练使用 109 条本地复核样本，以及 JD dataset 派生的 42,000 条外部类别训练样本。为降低领域偏移，本地样本在类别训练阶段执行 50 倍领域重采样。另有 7,000 条外部样本只用于评估。

## 2. 风险判断

随机森林使用类别与 8 个数值/布尔特征：已使用天数、剩余百分比、每周使用次数、使用人数、是否共享、是否有保质期、距离到期天数和是否破损。

最终结果不是直接照搬模型输出，而是使用混合决策：

- 30 天内到期进入过期/损坏风险。
- 低库存且高频使用优先建议补货。
- 随机森林负责正常与需要关注等模糊边界。
- 无有效期的破损标记会校准为需要关注，避免仅凭单个布尔字段直接给出最高风险；用户仍会看到破损原因并被建议人工检查。
- 模型文件缺失时自动降级为规则基线。

外部商品数据没有风险标签，因此完全不参与风险模型训练。

## 3. 评估结果

| 指标 | 原方案 | 优化方案 |
|:---|---:|---:|
| 真实留出类别 Accuracy（10 条） | 1.000 | 1.000 |
| 外部留出类别 Accuracy（7,000 条） | 0.226 | 0.979 |
| 外部留出类别 Macro F1 | 0.173 | 0.979 |
| 真实留出风险 Accuracy（10 条） | 0.600 | 0.900 |
| 真实留出风险 Macro F1 | 0.517 | 0.889 |

风险留出集样本很少，0.900 只能证明当前回归集上的改进，不能代表生产泛化能力。逐条预测、混淆矩阵和数据边界见 `reports/model_eval.md`。

## 4. 工程能力

- CSV 自动尝试 UTF-8、GB18030、GBK 编码，另支持 `.xlsx`。
- 单次批量输入最多 500 行；空行和非法数值有明确拦截。
- 反馈使用文件锁持久化，只生成待人工复核的重训候选。
- 模型元数据记录训练版本、样本规模、依赖版本和外部来源。
- 自动化测试覆盖特征解析、类别规则、混合风险边界、外部数据映射、批量输入、反馈与 UI 行为。

## 5. 客观限制

- 外部评估和训练来自同一商品平台，不能等同于任意自然语言输入。
- 本地风险标签规模仍小，后续改进应优先增加独立、人工复核的风险样本，而不是继续导入无状态标签的商品标题。
- 系统不能替代对药品有效期、电器破损和安全隐患的人工检查。
