# 宿舍生活用品智能小助手项目介绍

本系统是一个针对宿舍生活场景设计的物品管理与状态预测系统。通过级联机器学习模型与规则引擎的混合架构，实现物品类别的自动归类以及生命周期/使用状态风险的定量化评估。本系统在交互界面上结合了校园绿设计美学与双主题响应，并设计了闭环的用户反馈持久化流程。

以下是关于该系统在算法选择、界面设计、核心功能与技术亮点方面的直白客观介绍。

---

## 一、 算法设计与选择 rationale

系统采用了双模型级联架构（Cascading Model Architecture），将“文本分类”与“数值/特征风险分类”分步处理。

### 1. 物品类别预测：TF-IDF + 逻辑回归 (Logistic Regression)
* **任务目标**：根据用户输入的“物品名称”和“用户描述”（短文本），将其分类到 7 个类别之一（清洁日用、洗漱用品、健康与补剂用品、学习用品、电子配件、文体娱乐、其他用品）。
* **为什么选逻辑回归**：
  * **高维稀疏特征的高效拟合**：短文本经分词与 TF-IDF 转换后形成高维稀疏矩阵。逻辑回归在线性可分或高维稀疏空间中表现极其稳定，计算复杂度低。
  * **强可解释性**：逻辑回归模型可以输出各类别的预测概率值（Probability），而非简单的硬分类结果，能够为前端可视化图表提供直接的数据支撑。
  * **小样本适用性**：项目初始训练数据较少（约 109 条），逻辑回归配合 L2 正则化能有效控制过拟合。
* **分词策略**：使用 `jieba` 进行中文精确模式分词，并设置 `min_df=2` 过滤低频词，增强特征的泛化能力。
* **样本不均衡处理**：设置 `class_weight="balanced"`，根据类别频数自动缩放权重，避免模型倾向于大类。

### 2. 状态风险预测：随机森林 (Random Forest)
* **任务目标**：根据物品的类别、已使用天数、剩余百分比、每周使用次数、使用者人数、是否共享、是否有保质期、距离过期天数、是否破损等 9 维特征，预测 4 类风险状态（正常、需要关注、建议补货、过期/损坏风险）。
* **为什么选随机森林**：
  * **混合特征处理能力**：输入的 9 维特征既有分类特征（如 `category` 需要 One-Hot 编码），也有数值特征（如 `remaining_pct`），且数值范围差异极大。随机森林作为树集成算法，天生不需要对特征进行复杂的标准化或归一化。
  * **捕获非线性交互关系**：风险判断通常是非线性的。例如，一个物品“剩余百分比低”（20%）是否需要补货，取决于“每周使用次数”。若是高频消耗品，则是“建议补货”；若是低频药品且未过期，可能仅是“需要关注”。随机森林的决策树组合能够自然捕捉此类交叉特征规则。
  * **鲁棒性强**：随机森林对异常值和单特征噪声不敏感，通过集成多棵树降低了模型的方差。

### 3. 混合级联与兜底降级机制 (Hybrid Fallback Engine)
* **设计痛点**：对于全新的、未曾出现在训练集中的冷启动物品（例如“哑铃”或输入了“坏了”等强特征文本），如果单纯依赖机器学习，可能因为置信度低而输出错误分类。
* **解决方案**：引入基于置信度阈值（Confidence Threshold）的双层兜底规则机制：
  1. **类别分类兜底**：当分类模型预测概率的最大置信度低于 `0.40`（对于“其他用品”低于 `0.30`）时，系统自动切换至中文敏感词与正则匹配规则（在 [features.py](file:///D:/OneDrive/Documents/ML%E5%B0%8F%E7%BB%84%E4%BD%9C%E4%B8%9A/src/features.py) 中定义，如识别到“纸巾/垃圾袋”直接归为清洁日用，“笔/纸”归为学习用品），以保障基础分类的绝对准确。
  2. **风险预测兜底**：如果后台模型未训练或加载失败，系统将自动降级为规则基线引擎（Rule Baseline），通过“剩余比例与消耗速度的物理计算（预计剩余可用天数）”以及“强特征关键词识别（包含‘坏了’、‘过期’、‘失效’等直接判断为过期/损坏风险）”完成状态预测。

## 二、 页面设计与审美

界面设计避开了传统的通用灰色调，围绕校园生活场景进行了定制化美学设计。

* **校园绿 (Campus Green) 设计系统**：以校园植物的墨绿（`#4f7d55`）、柔和浅绿（`#edf6ea`）和米黄（`#fbf8ee`）为主色调，烘托宿舍与校园生活的温馨感。
* **双主题响应式支持 (Dark/Light Themes)**：系统在后台构建了包含 40 余个 CSS 自定义变量的机制。当用户在浏览器或 Streamlit 中切换浅色/深色模式时，主页背景、卡片容器、输入框、以及上传区域等组件的色彩、边框阴影和透明度会同步切换，提供一致的对比度和暗光保护。
* **卡片式布局与微阴影**：通过 CSS 的 `backdrop-filter: blur` 和 `rgba` 半透明属性实现轻量的磨砂玻璃（Glassmorphism）效果。表单与展示板块具有微小的悬停（hover）放大及阴影过渡，提升交互的层级感。
* **状态语义色设计**：对四个级别的风险卡片采用了高对比度、低饱和度的语义背景色：
  * **正常**：绿色背景，表达安全、稳定。
  * **需要关注**：黄色背景，提示潜在消耗。
  * **建议补货**：橙色背景，提醒库存紧张。
  * **过期/损坏风险**：红色背景，警告需要立即处理。
* **动态数据可视化**：
  * 使用 **Altair** 生成响应式水平条形图，展示模型对 7 个类别以及 4 类风险的概率分布（置信度百分比），让用户能够直观看到模型预测时的“犹豫度”或“肯定度”。
  * 批量预测结果使用环形图（Pie Chart）直观呈现各类风险用品的占比，辅助全局决策。

## 三、 功能架构与技术亮点

系统是一个具备数据闭环和工程化验证的完整应用。

### 1. 双模式预测功能
* **单件预测 (Single Mode)**：
  * 提供了快捷示例卡片（如“抽纸”、“蛋白粉”等常用配置），用户一键即可自动填充复杂的参数。
  * 预测结果页输出详细的 **“成因解释”**（解释是由于使用天数、剩余百分比、到期时间还是损坏引发的风险）和 **“针对性建议”**（如：尽快购买、注意避光保存等）。
* **批量清单 (Batch Mode)**：
  * 支持下载 CSV 导入模板。
  * 支持直接在网页的交互式表格（`st.data_editor`）中增加、修改或删除行，免去用户必须在本地编辑文件的麻烦。
  * 支持一键上传本地的 Excel 或 CSV 文件，并在表格内高亮展示每一行用品的预测结果，支持将最终的带风险评估的清单一键导出（支持 CSV 和 Markdown 文件下载）。

### 2. 闭环的用户反馈与纠偏机制
* **痛点**：机器学习模型不可避免会有误判，如何收集真实世界的数据进行迭代？
* **功能实现**：在单件预测结果下方，设计了“结果纠偏反馈表”。用户如果发现分类不准或风险有误，可以直接在此处选择正确的类别/风险，并提交纠正。
* **工程设计**：
  * 反馈模块使用 `filelock` 文件锁机制，保障在多线程/多并发网页访问下，写入 feedback 数据时不会发生写入冲突或文件损坏。
  * 离线脚本 `src/merge_feedback.py` 可将用户的纠错日志合并整理，剔除无意义条目后，自动导出为 `feedback_retrain_candidates.csv`，作为下一轮模型训练的候选集。

### 3. 数据鲁棒性与异常拦截
* **文件编码自适应**：批量导入 CSV 时，系统会自动尝试 `utf-8-sig`、`gb18030` 和 `gbk` 等多种中文常用编码格式，有效解决 Windows 系统下 Excel 另存为 CSV 导致的中文乱码问题。
* **空值与脏数据清洗**：在 [predict.py](file:///D:/OneDrive/Documents/ML%E5%B0%8F%E7%BB%84%E4%BD%9C%E4%B8%9A/src/predict.py) 中，模型输入采用严格的类型安全拦截（如 `_to_int`、`_to_float`、`_to_bool` 转换器），过滤非数值型脏数据，对于空行或超出上限（单次限制 500 行）的数据予以友好拦截。

### 4. 自动化测试与工程完整度
* 项目包含一个完整的自动化测试套件（共 24 个测试用例，运行于 `pytest`），对分类兜底逻辑、天数估算准确度、批量上传的最大限制、纠偏持久化文件锁、CSS样式作用域等边界和核心业务代码进行了高覆盖度的回归测试，保证了代码修改后的系统稳定性。

## 四、 总结与客观局限性

这是一个小巧但功能完整、遵循现代软件工程规范的机器学习原型项目。

* **优势亮点**：界面美观、双模型串联、支持冷启动降级兜底、配备了数据反馈环路与全套单元测试。
* **客观局限性**：由于当前的原始训练数据规模（109条）较小，模型的泛化性能和对罕见物品分类的精度还有提升空间，状态预测的准确度受限于特征工程的丰富程度。未来需要依靠用户反馈数据进行多轮滚动迭代。
