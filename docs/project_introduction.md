# 宿舍生活用品智能小助手：技术介绍

项目将宿舍用品管理拆分为两个步骤：先识别物品类别，再根据库存、消耗、有效期和破损状态判断风险。前端使用 Streamlit，支持单件预测、批量清单、结果解释和反馈记录。

## 1. 类别识别

类别管线包括：

1. 字符 `2-5 gram` TF-IDF，用于处理品牌、缩写和未登录词。
2. `jieba` 中文词组 `1-2 gram` TF-IDF，用于保留词组信息。
3. 卡方特征筛选与逻辑回归，输出 7 个类别的概率。
4. 对纸巾、药品、数据线等边界较明确的关键词执行领域规则，其余输入由模型判断。

训练数据包含 109 条本地人工复核样本，以及从 JD dataset 派生的 42,000 条商品类别样本。类别训练时对本地样本执行 50 倍领域重采样。另有 7,000 条同源商品样本用于评估；这些样本与训练数据来自同一公开数据源，标签由同一套规则映射产生，不是独立人工标注。

## 2. 风险判断

随机森林使用物品类别和 8 个数值或布尔特征：已使用天数、剩余百分比、每周使用次数、使用人数、是否共享、是否有保质期、距离到期天数和是否破损。

最终结果结合模型与规则：

- 30 天内到期时提示过期或损坏风险。
- 低库存且高频使用时提示补货。
- 随机森林处理其余边界情况。
- 无有效期物品只有通用破损标记时先提示关注，并要求用户人工检查。
- 模型文件缺失时使用规则基线，页面仍可返回结果。

外部商品数据没有风险标签，因此不参与风险模型训练。

## 3. 评估说明

| 评估集 | 样本量 | 类别 Macro F1 | 说明 |
|:---|---:|---:|:---|
| 同源规则映射留出集 | 7,000 | 0.979 | 用于检查当前清洗、映射和分类流程；不代表任意宿舍输入的实际准确率 |
| 本地人工复核留出集 | 10 | 见详细报告 | 样本较少，只用于回归检查 |

风险模型的 10 条本地人工复核留出结果保留在 `reports/model_eval.md`，不用于对外推断普遍泛化能力。

## 4. 应用实现

- CSV 自动尝试 UTF-8、GB18030 和 GBK 编码，并支持 `.xlsx`。
- 单次批量输入最多 500 行；空行、非法数值和越界输入会被拦截。
- 反馈通过文件锁写入日志，只生成待人工复核的重训候选。
- 模型元数据记录训练时间、样本规模、依赖版本和外部数据来源。
- 自动化测试覆盖特征解析、类别规则、风险边界、外部数据映射、批量输入、反馈和页面逻辑。

## 5. 当前限制

- 公开商品标题与宿舍用品输入存在领域差异。
- 同源规则映射评估集不等同于独立外部验证。
- 风险数据规模较小，应用结果仍需结合包装信息和物品实际状态人工核验。
- 项目不进行在线自动学习，反馈需人工复核后再离线重训。
