# AI 辅助开发记录

项目使用 AI 工具辅助需求梳理、代码编写、测试补充、界面调整和文档整理。数据范围、标签规则、评估口径和最终结果由开发者检查。

## 使用范围

| 阶段 | AI 辅助内容 | 人工检查内容 |
|:---|:---|:---|
| 需求与任务拆分 | 将宿舍用品管理拆分为类别识别和状态风险判断 | 确认功能边界与课程要求 |
| 数据处理 | 提供清洗、去重、抽样和隐私处理代码建议 | 检查字段、标签来源和脱敏结果 |
| 模型实现 | 辅助实现 TF-IDF、逻辑回归、随机森林和规则基线 | 比较评估结果并确认使用限制 |
| 外部数据 | 辅助检索和接入公开商品标题数据 | 核对许可证、固定版本和使用范围 |
| 应用开发 | 辅助实现 Streamlit 页面、批量处理和反馈记录 | 运行功能测试并检查异常输入 |
| 文档整理 | 辅助生成说明结构和表格 | 核对数字、术语和公开表述 |

## 数据与标签约束

- 本地训练数据为 109 条人工复核样本；本地人工复核留出集为 10 条。
- 外部 JD dataset 派生数据只用于商品类别识别，不用于风险模型。
- 外部数据按同一套规则映射类别，42,000 条用于训练，7,000 条用于流程评估。
- `label_source=rule_initial` 的本地样本不能直接进入训练集。
- 用户反馈只进入候选日志，人工复核后才能用于离线重训。

## 未采用的建议

- 未使用外部商品标题推导库存、有效期或破损标签。
- 未加入在线自动训练，避免少量反馈直接改变模型。
- 未接入大模型接口作为预测依赖，应用可在固定本地模型下运行。
- 未把同源规则映射评估结果描述为独立外部验证或实际准确率。

## 当前结果

- 类别识别使用字符与中文词组 TF-IDF、特征筛选、逻辑回归和领域规则。
- 风险判断使用随机森林与临期、低库存等决策规则。
- 页面支持单件预测、CSV/Excel 批量处理、结果解释和反馈记录。
- 自动化测试覆盖特征处理、模型加载、批量输入、反馈和页面逻辑。

AI 可以提高实现和整理效率，但不能替代数据来源核对、标签复核、运行测试和结果解释。项目公开文档对同源数据、样本规模和评估限制做了明确说明。
