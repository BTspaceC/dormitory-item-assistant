# 真实样本采集与冻结评估协议

## 采集目标

- 20 名匿名参与者，每人填写 3 件当前真实拥有的宿舍物品。
- 只保留经过人工复核的 60 条有效样本。
- 不采集姓名、电话、学号或联系方式。

## 防偏差规则

1. 参与者先独立填写类别、实际处理动作和原因，再运行模型预测。
2. 同一参与者的样本只进入一个数据分区：P001-P014 为训练集，P015-P020 为冻结测试集。
3. `user_feedback_log.csv` 中的演示记录和重复反馈不得自动进入训练集。
4. 复核人员不得依据模型预测修改标签；只依据参与者描述、数值字段和实际处理动作确认标签。

## 使用方法

1. 将 `宿舍用品真实样本采集模板.xlsx` 发给参与者，或按相同字段搭建腾讯问卷。
2. 汇总后填写复核列：`review_status`、`reviewer_category`、`reviewer_risk`、`review_notes`。
3. 运行：

```bash
python -m src.import_real_samples --input "采集结果.xlsx"
```

4. 检查 `data/collection/validated/` 下的有效、训练、冻结测试和退回记录。
5. 只有当 `collection_status=complete` 时才进入正式重训。

## 模型上线门槛

- 新冻结测试集 Macro F1 同时超过当前模型和规则基线。
- 四个风险类别都必须有非零召回率。
- 若门槛未达到，保留当前线上模型并继续采集，不通过改测试集或删除难例抬高分数。
