# 宿舍生活用品智能小助手

[在线体验](https://dormitory-item-assistant.streamlit.app/) · [项目仓库](https://github.com/BTspaceC/dormitory-item-assistant)

本系统是针对宿舍生活场景设计的物品管理与状态预测系统。基于真实用户访谈样本和人工复核扩展数据，实现物品自动分类与状态风险评估，辅助用户及时掌握补货、过期或损坏状态。

系统采用 Streamlit 搭建交互界面，核心业务流如下：
```text
输入/上传物品信息 -> 类别自动分类 -> 状态风险评估 -> 生成建议与成因 -> 记录用户反馈
```

## 一、系统核心功能

- **单件物品预测**：实时分析物品的所属类别、状态风险，并提供量化的判断依据与补货建议。
- **批量清单处理**：支持手动编辑或批量上传 CSV/Excel 清单，一键评估宿舍用品状态。
- **结果可视化展示**：直观呈现单件预测概率分布、批量风险比例和类别构成。
- **用户反馈持久化**：支持用户纠正分类和风险结果，保存反馈数据以用于模型演进。
- **自动化测试套件**：基于 `pytest` 覆盖核心规则、批量导入容错及反馈合并等模块。
- **项目集成说明页**：在系统内部直观查阅技术文档、模型评估报告与交付记录。

## 二、模型架构设计

系统设计了两个机器学习模型以实现级联预测：

| 预测任务 | 选用算法 | 输入特征 | 输出标签 |
|:---|:---|:---|:---|
| **物品类别分类** | `TF-IDF (char ngram 2-4) + Logistic Regression` | 物品名称、用户描述 | 清洁日用、洗漱用品、健康与补剂、学习用品、电子配件、其他用品 |
| **状态风险预测** | `RandomForest Classifier` | 类别、已使用天数、剩余百分比、使用频率、保质期及破损状态等 | 正常、需要关注、建议补货、过期/损坏风险 |

## 三、环境部署

推荐使用 Python 3.12 或兼容版本。

### 1. 开发模式安装（推荐）
```bash
python -m pip install -e .[dev]
```

### 2. 生产运行依赖安装
```bash
python -m pip install -r requirements.txt
```

## 四、系统运行

确保 `models/` 与 `data/processed/` 目录中已生成模型和数据后，运行以下命令启动服务：
```bash
python -m streamlit run app.py
```

服务启动后，可通过侧边栏切换三大模块：`单件预测`、`批量清单` 及 `项目说明`。

## 五、数据导入规范

批量导入支持 `.xlsx` 与 `.csv` 文件。CSV 格式默认兼容 `utf-8-sig`、`gb18030` 与 `gbk` 编码以防止中文乱码。单次导入行数上限为 `500` 行。

建议上传清单包含以下列字段：

| 字段名称 | 必填 | 说明 |
|:---|:---:|:---|
| 物品名称 | 是 | 宿舍生活用品名称 |
| 剩余量百分比 | 是 | 取值范围：0 到 100 |
| 每周使用次数 | 是 | 用于估算消耗速度 |
| 物品描述 | 否 | 补充用途、状态、品牌等信息 |
| 是否有保质期/有效期 | 否 | 药品、食品及补剂等物品建议选择 |
| 距离过期天数 | 否 | 整数值，留空则按无有效期处理 |
| 是否破损 | 否 | 电子配件或易损件建议选择 |

## 六、重训与数据准备

使用项目内置的脱敏访谈数据集进行数据预处理和重新训练：
```bash
python -m src.data_prepare
python -m src.train_models
python -m streamlit run app.py
```

若需导入新的原始访谈表（`.xlsx` 格式）：
```bash
python -m src.data_prepare --input "你的访谈表路径.xlsx"
python -m src.train_models
python -m streamlit run app.py
```
*注：预处理过程中会自动过滤并脱敏所有个人联系信息，确保训练数据集的安全合规。*

## 七、自动化测试验证

使用以下命令运行自动化测试：
```bash
python -m pytest tests
```
测试用例覆盖以下业务：
- 类别预测兜底规则与敏感词（破损/失效）匹配。
- 剩余可用天数估算准确度。
- 批量上传的字段映射与多编码文件读取。
- 批量上传最大行数限制拦截。
- 用户纠偏数据整理与再训练候选集生成。
- 浅色/深色主题切换的 CSS 作用域。

## 八、反馈整理与再训练机制

系统记录的用户纠偏数据暂不进行实时在线学习，以保证基础模型的稳定性。可通过以下命令离线汇总用户反馈：
```bash
python -m src.merge_feedback
```
汇总后将生成候选重训样本集：
```text
data/feedback/feedback_retrain_candidates.csv
```
该样本集经由人工校验审核后，方可并入基准训练集重训模型。

## 十、系统局限性

- **样本规模**：真实用户样本量较小，评估指标主要用于流程验证。
- **级联误差**：物品分类与状态预测为级联架构，分类模型的误差可能会传导至状态评估。
- **特征泛化**：TF-IDF 基于字面特征，对于无重叠词汇的同义表述识别能力有限。
